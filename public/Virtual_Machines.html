Using a Virtual Machine

  1. Compare and contrast the terms host and guest operating system.

    2. Host and Guest Operating Systems are both fully-functional and usable Operating Systems located on the same machine. However, the Host Operating System is the OS native to the machine, and Guest Operating Systems are the instances of Operating Systems installed and operating simultaneously on the physical machine.

  2. Name two different types of virtual machine software suitable for Windows based computers.

    3. Professor Messer cites Microsoft Virtual PC (available from Microsoft) and VirtualBox (available from Oracle), both of which are free, as being suitable for Windows based computers.

  3. When you plan to run a VM on a Windows Laptop, what two resources are considered especially critical?

    4. In order to run Virtual Machines, you need to have plenty of RAM and disk space. This is because the Guest Operating Systems and the Host Operating System are sharing the resources of the physical machine. Memory and disk space is required for each OS.

  4. Why might you want to turn your installation disks into ISO files?

    5. You can take everything off of multiple installation disks and encapsulate them into one disk image file. This makes the process of accessing and installing what is on those disks much faster and easier.

  5. In a virtual machine context, what is a snapshot?

    6. A snapshot can be viewed as a "recovery point" for an Operating System, with its configuration and current state saved. This means that you can take a snapshot of an OS, make changes to it, and even delete it, then go back to the the configuration the OS was in at the time the snapshot was taken.

Virtual Desktops and Servers

  1. What is a Hypervisor?

    2. A Hypervisor is a piece of software, also known as a "virtual machine manager", that allows you to manage the Virtual Machines and their relationships to the hardware in the physical machine. It also maintains separation between the virtual systems.

  2. What is a Type 1, Bare Metal Hypervisor?

    3. With a Type 1 Hypervisor, you start with a raw (blank) server and install the Hypervisor directly onto the machine. You then install the Operating System instances onto the hypervisor from another networked computer with Management Software using a Management Console.

  3. What is a Type 2 Hypervisor?

    4. Type 2 (or "hosted") hypervisors are installed onto an Operating System itself on a physical computer. You can then install Operating System instances into that hypervisor.

  4. Name three advantages of virtualization.

    5. By consolidating multiple servers onto one physical device, you can save a lot of room in the data center, have much more flexibility around what you can do with the hardware, and save on the cost of running one virtualization server vs. many physical servers.

Virtual Switches

  1. What impact does changing from a physical environment to a virtual environment have on the network components?

    2. When you migrate from a physical environment to a virtual environment, you are also virtualizing the networking components, removing them from the server platform.

  2. Describe a challenge associated with virtualizing your network components.

    3. One of the challenges is the integration with the outside world. Multiple servers may previously have each had its own ethernet connection, but you may have to build in connection redundancy in order for all of the virtual servers to share the network resources without compromising speed.

Network as a Service

  1. Explain what the term Network as a Service (NaaS) means.

    2. Network as a Service refers to network virtualization inside the cloud, completely offsite. You can then move your platforms, applications, and Network onto the cloud, completely virtualizing those operations.

  2. Name four advantages of Network as a Service (NaaS).

    3. - Allows you to have an application used by thousands of people and have the third party NaaS provider handle network traffic.

    - You can migrate your entire network around the world to accommodate for different geographic peak times.

    - Your network connection can expand and contract, allowing you to add or remove bandwidth as needed. This is useful when your networked application has varying time-based peak times.

    - You can work on your application from anywhere. If you decide to move your entire office, the network stays where it is.

Onsite vs. Offsite Virtualization

  1. Name three advantages of onsite virtualization

    2. - Complete control

    - Flexibility to change and shift as needed

    - Secure as you need

  2. Name three disadvantages of onsite virtualization.

    3. - Costly

    - Requires significant networking infrastructure

    - Not easy to upgrade

  3. What does the term "Virtualize in the cloud" mean?

    4. "Virtualize in the cloud" refers to offsite virtualization, where all of your applications, servers, and Operating Systems are maintained elsewhere.

  4. Name four advantages of offsite virtualization.

    5. - No infrastructure cost

    - Management/maintenance is handled by others

    - Geographical flexibility

    - Seemingly unlimited upgrade options

  5. Name two disadvantages of offsite virtualization

    6. - Data is also in the cloud (security concern)

    - Contractual limitations (long-term contracts)
